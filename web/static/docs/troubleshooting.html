PlotterForge · Recovery

Start with the symptom

PlotterForge keeps the last good geometry when it can and puts the actual failure message in the status bar. Read that message first; then use the smallest recovery that matches what you can see.

Protect the work first: do not delete a layer to fix a generation error. The previous SVG remains available after a failed regeneration. Save a Version before changing page, pen, or clipping settings during diagnosis.

Import and image orientation

SymptomLikely causeAction
Unsupported fileThe selected file is not a readable PNG, JPEG, BMP, TIFF, WebP or SVG.Open it in an image editor and export a normal RGB/RGBA PNG or JPEG. For vectors, confirm the file is actual SVG rather than a renamed PDF.
Photo looks rotated or squeezedOld project data may predate EXIF orientation normalization, or the file has unusual orientation metadata.Re-import the original photo. Current imports bake EXIF orientation into the pixels so preview, layer bounds and path finding agree.
Image is cropped immediatelyThe layer was set to Area → Fill, or the Drawing Area scaling mode is Crop to fill.Select the raster layer and choose Area → Fit. Fit preserves the complete image; Fill deliberately covers the padded area.
Transparent background produces marksThe file may be flattened against white or the selected module does not ignore near-white opaque pixels.Confirm the source has real alpha. For point samplers enable Ignore White; for strict boundaries use a Region or Mask.

Path finding and generation

SymptomLikely causeAction
Empty outputThe source is nearly white, Ignore White removed every candidate, a Region is empty, or scale/threshold settings suppress all marks.Switch Display to Both, Region to Whole image, Brightness/Contrast to 1, and restore the module defaults. Confirm a draft appears before reintroducing the Region.
Dense or slow outputPoint density, Shape Dither columns, curve steps, growth iterations or a composite multiplied the geometry.Cancel, halve the controlling density, cap point counts, or reduce columns. Draft on a thicker Pen width, then restore the measured pen before the final render.
Layer says staleA parameter changed after the last final render, or the visible result is only a live draft.Click Apply / Regenerate. A final result says clean; drafts intentionally remain stale.
Changing generator does nothingGenerator switching never overwrites automatically.Press ✦ Generate. If the selected layer already contains generation, choose New layer or explicitly confirm overwrite.
Custom style is missingThe Cavalry package or uploaded shape is not installed in this PlotterForge workspace.Rebake or upload it, then reopen the style picker. Existing cached layer geometry remains exportable even when its custom generator is unavailable.

SAM and acceleration

SymptomLikely causeAction
SAM setup incompleteSAM2, Torch, torchvision, or the selected checkpoint is absent.Quit PlotterForge and rerun setup-windows.bat or ./setup-macos.command. The server never installs Python packages while running.
Checkpoint is downloadingYou selected a SAM model whose checkpoint is not local yet.Keep PlotterForge open and watch the percentage. Do not start repeated downloads by changing models back and forth.
GPU fallbackCUDA/MPS is unavailable or a module uses a CPU-only stage.Read the backend badge. If setup previously verified the GPU but the badge changed, restart PlotterForge; otherwise rerun setup and inspect its hardware check. CPU output is valid, only slower.
Computer becomes unresponsiveThe working raster or geometry count exceeds comfortable memory.Cancel the job, reduce image dimensions or density, enable Drawing Area rescaling for drafts, and close other GPU-heavy applications.

Composition and export

SymptomLikely causeAction
Raster and paths do not alignA legacy layer, stale render, or mismatched transform is being compared.Choose Both, set rotation to 0 and scale to 100%, then regenerate. Raster layers generate in their own local coordinate system so a current result moves with the image.
Clipped outputThe layer extends beyond the page, a crop/mask is active, or Drawing Area clipping trims geometry.Turn on bounds, inspect the selected layer, reset Crop/Mask, and use Fit. Preview may show off-page overflow, but export and plotting intentionally clip it to the paper.
Wrong pen receives a strokePen distribution changed, or an imported SVG stroke colour is closer to another enabled pen.Inspect enabled pen colours and order. Explicit Inkscape layer labels win; unlabelled imported strokes are matched live to the nearest enabled pen colour.
Export appears to hangComplex clipping or occlusion is composing many layers.Watch per-layer export progress. Cancel safely, hide unnecessary layers, simplify masks, or export layers separately to identify the expensive layer.

Machine connection and movement

SymptomLikely causeAction
Serial port will not openWrong port, plotter powered off, driver/permission issue, or another process owns the device.Power the plotter, reopen Setup, select the current port, and close other plotter software. On a network bridge, verify the host is reachable before retrying.
Bridge conflictThe Inkscape/Tailscale serial bridge and PlotterForge both want the same physical device.Stop the bridge before direct PlotterForge plotting, or configure PlotterForge to use the bridge socket. Only one process can own the serial port.
Homing moves the wrong wayMachine firmware or coordinate orientation does not match the intended profile.Stop immediately. Do not compensate with artwork placement. Verify the plotter profile, limit switches and firmware direction before another Home.
Pen drags during travelPen up height/rate is too low, or the mechanism has play.Use Manual over scrap to raise Pen up in small increments. Cycle repeatedly before running a job.
Dots become blobsPen down height, lowering speed or settle delay is excessive.Calibrate Pen up and down heights on scrap paper, then shorten the down delay. Change one value at a time.
Jam or lost stepsPhysical obstruction, excessive speed, loose paper, or motor overload shifted the real head position.Press Stop so the pen lifts. Clear the obstruction, keep the paper fixed, Home, and verify origin. Resume only if the physical coordinate system still matches the saved job.
Stop / Resume lands incorrectlyThe paper moved, the machine lost its origin, or auto-rotation/setup changed after stopping.Do not resume onto valuable paper. Re-establish the exact sheet and origin first; otherwise discard the checkpoint and restart.

Collect diagnostics

Useful diagnostics are specific and reproducible. Before reporting a problem, collect:

  • The exact red status-bar message—hover to read the full text.
  • Operating system, GPU model, backend badge, plotter model, connection type, and pen width.
  • The smallest project, source image and exact parameter change that reproduces the symptom.
  • Whether the problem affects viewport only, exported SVG, Plot Preview, or real motion.
  • For startup/setup failures, the final terminal lines from the platform setup or launcher.

Remove private artwork if needed, but do not paraphrase the error. A literal message plus a minimal project is far more useful than a large screenshot of the whole desktop.
