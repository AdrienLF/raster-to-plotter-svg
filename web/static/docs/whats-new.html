A more physical image workflow

Recent changes make imported photographs behave like real composition objects and add a new symbol-based path-finding language. This page explains what changed, not every commit.

Transformable raster layers

Importing a PNG, JPEG, BMP, TIFF or WebP now creates a selected raster layer. The complete image is fitted into the drawing area without cropping, and it owns the pixels used by its Path Finding settings.

  • Move, scale and rotate the image independently.
  • Use Fit and Fill to preserve the whole image or cover the padded drawing area.
  • Choose Raster, Paths or Both without losing the source.
  • Generated strokes remain in the layer’s local space, so they follow every later transform.
  • Off-page overflow remains visible while composing, then clips to the page for SVG export and plotting.

Shape Dither

Shape Dither quantizes image tone into a grid of symbols. Built-in circles, squares, stars, triangles, crosses and rounded squares can change scale with tone and rotate to the local image gradient in 90° steps.

  • Two to 32 tone levels, with optional Floyd–Steinberg error diffusion.
  • Original-aspect or centred square sampling.
  • Independent shape and export-only background colours.
  • Direct SVG upload for a single scalable state.
  • Cavalry baking for up to 32 real vector states with optional source colours.

EXIF orientation

Phone and camera orientation is now baked into imported pixels. The browser preview, layer box, path-finding raster and saved project all see the same width, height and aspect ratio instead of relying on different interpretations of EXIF orientation.
