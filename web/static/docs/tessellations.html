Bake a pattern in Cavalry

Draw one repeat unit in Cavalry, choose which numeric parameters should react to tone, and bake it into a reusable Plotter Studio tessellation. Studio then tiles those vector states across any image and chooses the light or dark version from the image tone.

One repeat unit

Make the tile you want repeated, not a whole page. The active Cavalry composition is the tile bounds.

32 tone states

The bridge renders 32 SVG snapshots from light to dark. Studio picks between them per tile.

The idea

A tessellation is a pattern that repeats on a lattice. In this workflow, Cavalry draws the vocabulary of one tile and Plotter Studio decides where each tile lands on the page. Your source image supplies the tone: light areas use the lightest baked states, dark areas use the darkest states, and midtones use the states in between.

The most useful setup is a small Cavalry composition containing one tile and one or more selected numeric attributes. For each selected attribute you set a Light value and a Dark value. The bridge moves every linked value together across 32 steps, renders each step as SVG, and installs the result as a Studio style.

Workflow

  1. Start Plotter Studio. The bridge sends everything to the local server at http://localhost:7438.
  2. Install the bridge in Cavalry. In Cavalry, open Help → Show Scripts Folder, copy cavalry/plotter-bridge.js there, then run it from Window → Scripts → plotter-bridge.
  3. Prepare one repeat unit. Work in the active Cavalry composition. Use a disposable or clean composition when testing; the tile bounds come from that composition's resolution.
  4. Name the pattern. In the bridge's Tessellation section, fill in Name. Use a visible name between 1 and 80 characters.
  5. Bind a parameter. Select exactly one numeric Cavalry attribute, then click Add selected parameter. The bridge adds a row with Light and Dark values.
  6. Set Light and Dark. Light is the value used by the lightest image tones; Dark is the value used by the darkest tones. Add more selected parameters if the tile should morph in several ways at once.
  7. Choose the lattice. Pick Rectangular, Brick, Hex/Isometric, or Custom. Set W and H, or the custom A/B vectors.
  8. Bake it. Click Bake tessellation. The progress bar counts from 1 to 32 while Cavalry renders the states. Your original Cavalry values are restored after success or failure.
  9. Use it in Studio. Add or edit a path-finding layer, choose the new style in the Tessellation family, tune the Studio controls, and click Apply / Regenerate.
The Cavalry bake creates the pattern language. Studio controls remain live, so you can change scale, rotation, phase, tone response, inversion and duplicate cleanup without rebaking.

Lattice presets

PresetWhat it doesGood for
RectangularRepeats the tile in straight rows and columns.Grids, wallpaper repeats, simple mechanical patterns.
BrickOffsets each row by half the tile width.Masonry, woven marks, scales, staggered organic rhythms.
Hex/IsometricUses a half-width offset and an isometric-height second axis.Triangular rhythms, hex-like texture, dimensional tiling.
CustomUses your custom A and B repeat vectors.Slanted repeats, experimental lattices, precise Cavalry layouts.

Studio controls

ControlWhat to adjust
ColumnsHow many repeats fit across the page width. Raise it for smaller, denser tiles.
RotationRotates the whole lattice around the page.
Phase X/YSlides the pattern along the first and second lattice axes, useful for lining a motif up with the image.
Tone ResponseBends the source-image tone before state selection. Higher values hold onto lighter states longer.
Invert ToneSwaps the light and dark ends of the baked state range.
Remove Duplicate LinesRemoves shared tile edges and welds the rest into longer strokes for cleaner plotting.

Rebaking safely

Rebake when you need to change the Cavalry artwork, selected attributes, Light and Dark endpoints, or lattice vectors. Reusing the same pattern name replaces the installed style. Already generated layers keep their cached SVG, so you can compare an old result with a new bake by saving versions before you regenerate.

Your original Cavalry values are restored at the end of the bake, even when the server rejects a state or the bridge reports an error. If the script says the server is offline, start Studio and bake again; if a package is missing later, reinstall or rebake before regenerating layers that use it.

Troubleshooting

ProblemWhat to do
Selection errorSelect exactly one Cavalry attribute before clicking Add selected parameter. The bridge reads the active selection, not the layer list.
Non-numeric attributeOnly finite numeric attributes can be baked. Choose a numeric control such as position, rotation, scale, radius, opacity or a numeric effect parameter.
Invalid vectorsUse finite lattice values. For Custom, A and B must point in different directions; collinear vectors cannot tile a plane.
Server failureConfirm Studio is running on port 7438. If an upload fails, the bridge restores Cavalry values and you can bake again.
Missing styleOpen the Path Finding style browser and look under Tessellation. If it is not there, rebake and watch for the bridge's installed message.
Missing-package regenerationExisting cached SVG can still appear in an opened project, but regeneration needs the package under Studio's local tessellation store. Reinstall or rebake that pattern.

Smoke checklist

When you have real Cavalry available, use a disposable composition and check these five cases before relying on a new bridge or server build:

  1. One-parameter bake: bind one numeric attribute, use distinct Light and Dark values, and confirm the installed pattern changes with image tone.
  2. Multi-parameter bake: bind two numeric attributes and confirm both original Cavalry values return after the bake.
  3. Preset sweep: bake or preview Rectangular, Brick, Hex/Isometric and Custom vectors with visible tile offsets.
  4. Failure recovery: stop Studio or force a rejected state, then confirm Cavalry values are restored and no half-installed pattern appears.
  5. Studio regeneration: choose the custom Tessellation style, adjust Columns, Tone Response and Remove Duplicate Lines, then click Apply / Regenerate.
