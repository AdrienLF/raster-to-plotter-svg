Your first plot, in five minutes

  1. Import an image — File → Import image… or the 🖼 button on the left rail. A JPEG/PNG becomes a selected raster layer, fitted without cropping and ready to move, resize or rotate; an SVG becomes a vector layer.
  2. Apply path finding — click the imported layer's ⚙ (or ＋ Path finding for another layer). Pick a module — start with Voronoi Stippling — then hit Apply / Regenerate. The strokes stay aligned with the image as you transform the layer.
  3. Set your pens — in the Pens panel choose colour and stroke width (mm). The preview redraws with true widths; a flat nib even renders its calligraphic angle.
  4. Check the page — Drawing Area sets the paper size, margins and how the image fits. A4/A3 guides are drawn on the canvas.
  5. Plot — open the Plot step: connect the port in Setup, read the time estimate, position the drawing on the paper, press Start.

Getting around

ActionHow
ZoomScroll wheel — zooms toward the cursor.
PanDrag anywhere on the canvas background.
Fit pageFit button (top right) or the ⤢ rail button.

Rotate / Fit / FillUse Rotate ° in Layers, or Area → Fit / Fill to centre the whole layer or cover the padded drawing area.
Nudge placementThe X/Y readout in the toolbar shows the plot placement in mm.
Stop anythingThe red ■ on the rail halts processing or plotting.
Undo⌘Z / Ctrl+Z (or Edit → Undo) restores the composition after a layer delete or an unwanted regenerate — up to ten steps back.

Projects & versions

Everything lives in a project: the source image, layers, regions, painted field masks, pen sets and settings. The Project menu creates, renames, opens and deletes them; the current project autosaves as you work.

The Versions panel is your contact sheet: after a result you like, type a name and ＋ Save. Each version snapshots the whole visible page — every layer with its geometry and settings — plus a thumbnail, so you can compare directions and restore any of them later. Rate them, keep notes, stay honest about which experiment actually looked best.

Exporting

File → Export SVG writes the composed page as a single SVG with one Inkscape layer per pen — ready for any other tool. Export layers (zip) writes one SVG per composition layer plus a manifest, for when another program does the final assembly.

Under the hood (one paragraph)

Generation runs on a working raster where one pixel ≈ one pen width, so "density" always means what the pen can actually resolve. Heavy sampling runs on the GPU when available — the status bar shows the backend (torch-mps, cuda or numpy). Output is vector all the way down: mm-true SVG in, G-code out.
